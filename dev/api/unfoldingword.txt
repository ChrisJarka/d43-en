====== unfoldingWord API ======


Official specification for unfoldingWord API, version 1:


===== unfoldingWord App Catalog =====


    api.unfoldingword.org/obs/txt/1/obs-catalog.json


===== translationStudio App Catalog =====


    api.unfoldingword.org/ts/txt/1/ts-catalog.json


===== JSON Text =====


    api.unfoldingword.org/obs/txt/1/[langcode]/obs-[langcode].json


e.g. api.unfoldingword.org/obs/txt/1/en/obs-en.json


===== Audio =====


    api.unfoldingword.org/obs/mp3/1/[langcode]/obs-[langcode]-[frame].[format]


e.g. api.unfoldingword.org/obs/mp3/1/en/obs-en-01.mp3


===== Images =====


    api.unfoldingword.org/obs/jpg/1/[langcode]/[heightpx]/obs-[langcode]-[chapter]-[frame].[format]


e.g. api.unfoldingword.org/obs/jpg/1/en/360px/obs-en-01-01.jpg

e.g. api.unfoldingword.org/obs/jpg/1/en/2160px/obs-en-01-01.jpg


===== translationStudio Resource Catalog =====

<code>
  api.unfoldingword.org/ts/txt/2/catalog.json
</code>

This catalog records resources available for translation in translationStudio. In the following examples each catalog reference could be generated dynamically in order to slim down the api size.

Example:

<code>
location: /ts/txt/2/catalog.json
content:
[
  {
    "slug": "obs",
    "date_modified": "20141207",
    "lang_catalog": "https://api.unfoldingword.org/ts/txt/2/obs/languages.json"
  },
  {
    "slug": "luk",
    "date_modified": "20141207",
    "lang_catalog": "https://api.unfoldingword.org/ts/txt/2/luk/languages.json"
  }
]
</code>

The language catalog would contain the translated project name and description along with necessary information regarding the language and it's status. Optional meta in the project provides the ability to create soft/virtual categories when viewed in the app. e.g. Bible→New Testament→Luke.

The resource catalog would contain the different types of resources available for translation. When multiple resources are present the app will supply ui controls to switch between resources.
